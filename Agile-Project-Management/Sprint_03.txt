2. SPRINT OBJECTIVE

---

To implement a secure authentication system using JWT and enforce role-based
access control for different system users (Admin, Doctor, Receptionist).

By the end of this sprint, the system should:

- Allow user login and token generation (JWT)
- Protect API endpoints using authentication
- Restrict access based on user roles
- Enable secure communication between client and backend

3. USER ROLES

---

🔹 Admin:

- Full system access
- Manage users, appointments, and system data

🔹 Doctor:

- View assigned appointments
- Access patient information
- Limited modification rights

🔹 Receptionist:

5. IMPLEMENTATION DETAILS

---

🔹 STEP 1: INSTALL JWT SUPPORT

pip install djangorestframework-simplejwt

---

🔹 STEP 2: CONFIGURE REST FRAMEWORK (settings.py)

REST_FRAMEWORK = {
'DEFAULT_AUTHENTICATION_CLASSES': (
'rest_framework_simplejwt.authentication.JWTAuthentication',
),
}

---

🔹 STEP 3: ADD JWT URLS (backend/urls.py)

from rest_framework_simplejwt.views import (
TokenObtainPairView,
TokenRefreshView,
)

urlpatterns = [
path('admin/', admin.site.urls),

path('api/token/', TokenObtainPairView.as_view(), name='token_obtain_pair'),
path('api/token/refresh/', TokenRefreshView.as_view(), name='token_refresh'),

]

---

🔹 STEP 4: ROLE-BASED PERMISSIONS (appointments/permissions.py)

from rest_framework.permissions import BasePermission

class IsAdmin(BasePermission):
def has_permission(self, request, view):
return request.user.role == 'admin'

class IsDoctor(BasePermission):
def has_permission(self, request, view):
return request.user.role == 'doctor'

class IsReceptionist(BasePermission):
def has_permission(self, request, view):
return request.user.role == 'receptionist'

6. AUTHENTICATION FLOW

---

1. User sends login request:
   POST /api/token/

2. System validates credentials

3. System returns:
   
   - Access Token
   - Refresh Token

4. Client stores token

5. Client sends token in requests:
   Authorization: Bearer <access_token>

6. Backend validates token before granting access

---

7. PROTECTED API FLOW

---

Example:
GET /api/admin-area/

Rules:

- User must be authenticated
- User must have role = admin
- Otherwise access is denied (403 Forbidden)
